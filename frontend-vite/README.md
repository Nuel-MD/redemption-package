# Redemption Package

## Overview

The Redemption Package is a full-stack web application designed to provide users with access to various spiritual resources, including Bible verses, books, worship materials, and more. The application is built using React for the frontend, with Tailwind CSS for styling, and Node.js with Express for the backend. PostgreSQL is used as the database to store categories and items.

## Features

- **User-Friendly Interface**: A responsive design built with Tailwind CSS.
- **Dynamic Content**: Fetches and displays Bible verses, books, worship resources, and more from a backend API.
- **Admin Authentication**: Secure routes for admin functionalities using JWT authentication.

## Technologies Used

- **Frontend**: React, TypeScript, Tailwind CSS
- **Backend**: Node.js, Express, PostgreSQL
- **Database**: PostgreSQL
- **Deployment**: Vercel for frontend and backend

## Expanding the ESLint configuration

If you are developing a production application, we recommend updating the configuration to enable type aware lint rules:

- Configure the top-level `parserOptions` property like this:

```js
export default tseslint.config({
  languageOptions: {
    // other options...
    parserOptions: {
      project: ['./tsconfig.node.json', './tsconfig.app.json'],
      tsconfigRootDir: import.meta.dirname,
    },
  },
})
```

- Replace `tseslint.configs.recommended` to `tseslint.configs.recommendedTypeChecked` or `tseslint.configs.strictTypeChecked`
- Optionally add `...tseslint.configs.stylisticTypeChecked`
- Install [eslint-plugin-react](https://github.com/jsx-eslint/eslint-plugin-react) and update the config:

```js
// eslint.config.js
import react from 'eslint-plugin-react'

export default tseslint.config({
  // Set the react version
  settings: { react: { version: '18.3' } },
  plugins: {
    // Add the react plugin
    react,
  },
  rules: {
    // other rules...
    // Enable its recommended rules
    ...react.configs.recommended.rules,
    ...react.configs['jsx-runtime'].rules,
  },
})
```
